# پایشگر اجرای پردازش‌های لینوکس با eBPF

[Read in English](README.md)

این پروژه یک نمونهٔ آموزشی و سبک است که اجرای پردازش‌ها را در لینوکس با eBPF ثبت می‌کند و رویدادها را در یک داشبورد کوچک Flask نمایش می‌دهد.

هدف پروژه، نمایش نحوهٔ مشاهده‌پذیری هستهٔ لینوکس، انتقال رویدادهای eBPF به فضای کاربر و ارائهٔ آن‌ها در یک رابط وب ساده است. این پروژه برای محیط عملیاتی یا استفاده به‌عنوان سامانهٔ امنیتی کامل طراحی نشده است.

## معماری

```text
فضای هسته
  برنامهٔ eBPF متصل به sched:sched_process_exec
        |
        v
فضای کاربر
  دریافت رویدادهای perf توسط بارگذار Python و BCC
        |
        v
حافظهٔ موقت رویدادها
  نگهداری ۱۰۰ رویداد آخر
        |
        v
داشبورد Flask
  رابط فارسی با به‌روزرسانی خودکار هر ۲ ثانیه
```

## اطلاعات ثبت‌شده

- شناسهٔ پردازش (PID)
- نام دستور یا پردازش
- زمان ثبت رویداد

## پیش‌نیازها

- Ubuntu Desktop 24.04 LTS
- هستهٔ لینوکس نسخهٔ 6 یا جدیدتر
- Python 3
- Flask
- بسته‌های BCC / bpfcc
- دسترسی root برای بارگذاری برنامهٔ eBPF

> این پروژه مستقیماً روی Windows یا macOS اجرا نمی‌شود. برای این سیستم‌عامل‌ها از یک ماشین مجازی Ubuntu استفاده کنید.

## نصب

ابتدا مخزن را دریافت کنید:

```bash
git clone https://github.com/Hesam-deb/linux-ebpf-exec-monitor.git
cd linux-ebpf-exec-monitor
```

سپس اسکریپت‌های نصب و اجرا را آماده و وابستگی‌ها را نصب کنید:

```bash
chmod +x scripts/install.sh scripts/run.sh
./scripts/install.sh
```

اسکریپت نصب نسخهٔ هسته را بررسی می‌کند، بسته‌های BCC را با `apt` نصب می‌کند، یک محیط مجازی Python با دسترسی به BCC سیستم می‌سازد و Flask را نصب می‌کند.

## اجرا

```bash
sudo ./scripts/run.sh
```

سپس این آدرس را در مرورگر باز کنید:

```text
http://127.0.0.1:5000
```

برای توقف برنامه کلیدهای `Ctrl+C` را فشار دهید.

## آزمایش داشبورد

برنامه را در ترمینال اول اجرا کنید:

```bash
sudo ./scripts/run.sh
```

در ترمینال دوم چند دستور اجرا کنید:

```bash
ls
whoami
ip a
ping -c 2 8.8.8.8
```

داشبورد باید به‌صورت خودکار به‌روزرسانی شود و رویدادهای اجرای این دستورها را نمایش دهد.

## امکانات داشبورد

- تعداد کل رویدادهای دریافت‌شده
- تعداد دستورهای یکتا در ۱۰۰ رویداد نگهداری‌شده
- زمان آخرین به‌روزرسانی
- جدول زمان، PID و نام دستور
- رابط راست‌به‌چپ فارسی با فونت Vazirmatn

## محدودیت‌های آموزشی

- فقط اجرای پردازش‌ها ثبت می‌شود؛ فعالیت فایل، شبکه و احراز هویت پایش نمی‌شود.
- رویدادها فقط در حافظه نگهداری می‌شوند و با توقف برنامه از بین می‌روند.
- داشبورد به‌جای ارسال زنده، هر ۲ ثانیه صفحه را تازه‌سازی می‌کند.
- احراز هویت و کنترل دسترسی وجود ندارد.
- پروژه برای نمایش روی یک سیستم لینوکسی طراحی شده است.
